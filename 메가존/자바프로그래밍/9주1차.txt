    1. 클래스와 구조체의 차이
 + 멤버 변수 Member Variable(속성 Attribute)
 + 멤버 함수 Member Function(Method)
 + 정보 통합 서비스


    2. 은닉성(Encapsulation)
 + 객체 내부에서 정보 관리
  - 무결성 or 유효성
 + Member Function을 통해 서비스를 제공받음
  - 객체 기반의 호출
  - 객체 정보를 기반으로 작업을 수행
  - this: 호출될 때 기반이 되는 객체

    (1)접근 지정자
 + Privte
 + Default

    (2)논리 연산자 사용 시 주의 사항
 + 논리 곱, 합 연산 시 불필요한 경우,
  두 번째 피연산자 내의 연산은 수행하지 않음


    3. 생성자
 + 생성자(Constructor)는 멤버 함수와 유사함
 + 클래스와 동일한 이름 사용
 + 매개 변수를 가짐
 + 반환값이 없음
 + 오류 발생 처리 방법에 따라 객체가 생성되지 않을 수 있음

    (1)기본 생성자(Default Constructor)


    4. Method Overload
 + 동일한 이름의 멤버 함수(생성자 포함)가 공존할 수 있음
 + 매개 변수 구성의 차이로 구분함


    1. 은닉성
객체 내 정보의 유효성과 무결성 확보
데이터의 배타적 관리로 안전성은 확보하지만, 접근할 때는 반드시 멤버 함수를 거쳐야 함
객체 기반, 정보 기반의 서비스를 제공하기에 용이

    2. 생성자
객체를 생성할 때 자동으로 호출되는 일종의 멤버 함수
매개변수를 가짐
반환 값을 지정할 수 없음
객체 생성 실패 시 메모리에 객체는 생성되지 않음

    3. 메소드 오버로드
하나의 클래스에 동일한 이름의 멤버 함수가 다수 선언될 수 있음
매개 변수의 구성으로 구분
반환 값의 변수 형으로는 구분할 수 없음
예외 처리로 구분할 수도 없음
