    1. 네트워크의 필요성

    (1)Computer Network의 발전
목적: 원거리 컴퓨터간 데이터 교환(공유)
주체: 미국방성과 대학
 - 군사 및 학술 목적
 - ARPANET
 - 분산적 구조로 진행
 - 핵전쟁 등에서 생존토록 설계
 - 회선 중심 연결에서 패킷 중심 연결로 변화
 (패킷: 데이터를 분산해서 전송 > 복원)

    (2)인터넷의 탄생
 - 1989년 상업적 목적으로 사용 허가

    (3)인터넷과 산업
 - 컴퓨터의 등장
 - 모든 컴퓨터의 네트워크 접속
 - 사무 및 생산 자동화
 - 생활 전 분야에 걸친 인터넷의 침투
 *Internet of Things > Internet Communication Things

    
    2. LAN, WAN 그리고 Inter Net

    (1)LAN(Local Area Network, 근거리 통신망)
정의(IEEE)
 - 한정된 지역 컴퓨터를 기본으로 하는 여러 전자기기 사이의 정보교환
 - 구축한 사용자가 직접 관리, 운영
분류
 - 이더넷
 - 무선랜

    (2)WAN(Wide Area Network, 광역 통신망)
 - 국가, 대륙 또는 전 세계를 포괄
 - 공중망(Public Network)으로 통신사업자가 구축한 망 사용
특징
 - 네트워크 형태의 복잡
 - 스위치 사용
 - 장거리인 경우가 대부분, 비용이 상대적으로 높음


    (3)인트라넷
정의
 - 단체의 구성원만이 접근 가능한 사설망
 - 폐쇄적 근거리 통신망
보안
 - 방화벽을 설치하여 외부로부터의 접근 제한


    3. 네트워크의 구성 요소

    (1)리피터(Repeater)
 - 손상된 신호의 증폭
 - 장거리 통신시 필요

    (2)허브(Hub)
 - 다중 포트 리피터 역할
 - 한 포트로 받은 프레임을 모든 포트로 전송(멀티탭)
더미 허브
 - 연결된 컴퓨터간 스테이션들 간 데이터 중계
 - 대역폭 분활 사용 > 전송 효율 감소
 - 소규모 네트워크 환경에서 사용
스위칭 허브
 - 수신지 주소로 스위칭하는 기능 포함
 - 네트워크 효율 좋음
인텔리전트 허브
 - 지능형 허브, 네트워크 관리 시스템 역할

    (3)루터(Router)
 - 서로 다른 네트워크 사이 패킷의 경로선택 기능
 - 최근 스위치에서 루팅 기능도 수행

    (4)게이트웨이(Gateway)
 - 다른 프로토콜로 변환
 - LAN과 WAN 사이에 위치
 - 방화벽 기능 수행하도록 설계

    (5)스위치(Switch)
 - 계층별 스위치(OSI 7 Layer)
 - 로드 밸런싱(네트워크 조절)
 - 방화벽



    네트워크 명령어를 통해 네트워크 상태 확인

ipconfig (-all)
 - 모든 네트워크 확인
ping (Gateway Address)
 - 핑을 보내 게이트웨이 혹은 DHCP 서버 응답시간을 확인
tracert (Domain Address)
 - 어떤 루터를 거쳐가는지 경로를 보여줌
