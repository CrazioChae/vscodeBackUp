    (1)File Transfer Protocol의 목적
 + 컴퓨터 프로그램이나 데이터의 공유
 + 원격으로 컴퓨터를 사용할 수 있도록 함
 + 다수의 사용자들간의 파일 공유를 원활하게 함
 + 안정적이고 효과적으로 데이터 전송

    (2)파일 전송 프로토콜의 역사
 + 1985년 현재의 FTP로 대체 됨 / RFC 959
 + 1994년 방화벽 친화 FTP 수동 모드 활성화 / RFC 1579
 + 1997년 보안 모드 확장 제안
 + 1998년 IPv6 지원 추가, 새로운 수동 모드 추가

    (3)FTP의 보안
 + 보안 프로토콜로 계획되지 않았음
 + 1999년 RFC 2577에서 취약점 나열
  - 무차별 대입 공격
  - 패킷 가로채기
  - 포트 훔치기
 + FTP 바운스 어택


    1. FTP의 목적
데이터 공유
효율적인 데이터 전송 방식 제공
이 기종 간 데이터 공유 지원

    2. FTP 규약의 특성
명령어 연결과 데이터 전송 연결로 구분됨
데이터 전송은 능동 모드와 수동 모드로 가능함
두 개의 서버 간의 데이터 교환을 지원함
