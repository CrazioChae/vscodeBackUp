    1. OSI 7 Layer의 개념과 필요성

    (1)개념
 + 인터넷에 연결된 시스템간 통신 규약의 표준
 + International Organization for Standardization 에서 제정

    (2)필요성
 + 표준화된 규약의 필요
 + 모듈화를 통한 개발 및 검증의 효율을 높임
  - 모듈화를 계층별로 구현
   ex) API, 라이브러리 등
  - 각 계층별 기능을 정의


    2. OSI 7 Layer의 각 계층별 기능

    (1)물리 계층(Physical Layer)
 + 하드웨어적 전송 규약(물리적 신호)
 + 데이터의 오류 확인 등은 상위 계층에서 진행
 + 기능적 정의는 단순하지만, 정교한 설계가 필요

    (2)데이터 링크 계층(Data Link Layer)
 *+ point to point 간 신뢰성 있는 전송 보장
 *+ MAC Address를 통해 통신 주체간의 연결
 + 전기적 신호에 발생할 수 있는 오류 처리(수정 혹은 재전송 요구 등)
 + 데이터의 흐름 제어
 + 데이터 단위는 *Frame

    (3)네트워크 계층(Network Layer)
 + 라우팅 기능
  - 데이터를 목적지까지 빠르고 안정적으로 전송하는 기능(최적 경로)
 + 이 계층에서는 IP 주소를 사용하게 됨(*TCP/IP)
 + 데이터 단위는 *Packet

    (4)전송 계층(Transport Layer)
 + 통신의 활성화
 + 사용자들이 주고 받는 데이터의 신뢰성 확보
 + *오류 검층, 흐름 제어, 중복 검사
 + 전송 실패 시 *재전송 등도 수행
 + 데이터 전송을 위해 port 번호를 사용
 + 데이터 단위는 *Segment

    (5)세션 계층(Session Layer)
 + 통신 접속자들간의 동기화와 오류 복구 등의 명령들을 처리
 + 통신을 위한 세션을 확립, 유지, 중단을 수행(운영체제에서 수행)

    (6)표현 계층(Presentation Layer)
 + 데이터(코드)간의 번역을 담당
 + 데이터의 포장, 압축, 암호화 등을 수행

    (7)응용 계층(Application Layer)
 + 통신을 필요로 하는 *응용 서비스 구현
 + 네트워크 프로그램이 여기 해당


    3. TCP/IP

    (1)탄생
 + 미국 방위 통신청에서 개발 배포
 + 대부분의 패킷 통신은 TCP/IP로 구현되어 있음
 + OSI 7 Layer를 기반으로 정의된 규약

    (4)TCP와 IP의 기능 정리
 + IP 계층
  - IP 주소 기반의 연결 확보(Route)
  - 데이터 전달 여부는 관여하지 않음
  - 데이터의 전송 순서도 관여하지 않음
 + TCP(Transmission Control Protocol)
  - 데이터의 전송 확인
  - 연결 지향형 프로토콜

    (5)UDP(User Datagram Protocol)
 + 비연결지향
 + 오류처리를 하지 않음
 + 순서 재조합 기능도 수행하지 않음
 + 보내기만 할 뿐 수신 확인은 하지 않음
 + 실시간 멀티미디어 정보 처리를 위해 사용


    4. Layer별 Switch

    (1)L2 Switch
 + 스위칭 허브
 + MAC Address를 이용(특정 시스템에게 전송)
 + 단순한 구조를 가지며 신뢰성이 높음
 + 대역폭을 나눠 사용하게 됨

    (2)L3 Switch
 + IP 주소를 이용
 + 루터 기능을 수행

    (3)L4 Switch
 + Port 번호를 활용
 + 여러 대의 서버를 하나처럼 사용하며 Load Balancing을 할 수 있음
 + 상위 프로토콜에 의존적이며 복잡함

    (4)L7 Switch
 + 데이터의 내용을 확인하고 대처할 수 있음
 + 방화벽 등의 용도로 사용(대부분 방화벽 용도로 사용)


1. OSI 7 Layer
국제 표준 통신 규약
계층별 독립된 기능 수행
데이터의 효율적 안정적 전송을 목적으로 함

2. TCP / IP
인터넷에서 범용적으로 사용되는 통신 규약
SOCKET이라 불리우기도 함
UDP라는 실시간 통신을 위한 규약도 존재함
