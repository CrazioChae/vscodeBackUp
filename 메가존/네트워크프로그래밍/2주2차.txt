    1. 인터넷의 주소 체계
패킷: 받는사람의 주소와 보내는 사람의 주소
    (1)필요성
 - 네트워크에 연결된 시스템을 식별하기 위해 필요

    (2)종류
 + MAC(Media Access Control Address)
  - NIC(Network Interface Controller)에 주어진 주소
  - 하드웨어 주소(물리 주소)
  - 과거에는 고정이었으나, 최근에는 변경 가능
  - 다수의 NIC 시스템 가능, 각 NIC는 상이한 MAC 주소를 가짐
  - 특수 목적 네트워크 장비(멀티 레이어 스위치, 라우터 등)
   하나 이상의 영구적 MAC 주소를 갖기도 함
  - 6 octets: 8비트 6개, 앞 3개는 제조사의 식별 번호,
   뒤 3개는 제조사 별 기준에 의한 식별 번호

 + 참고
유니캐스트
 - 지정된 NIC에서만 전송된 정보(패킷)를 처리
멀티캐스트
 - 추가적으로 설정된 기준을 기반으로 전송된 정보를 처리할지 결정
브로드캐스트
 - 전송된 모든 정보를 처리

 + IPv4(인터넷 프로토콜 version 4) 주소체계
  - 32bit(4byte) 주소 체계(*MAC 주소는 48bit)
  - Host를 찾아가기 위해 사용(Routing)
  *MAC 주소는 고정이지만 IPv4 주소는 유동적
  - 현재는 IPv6 주소체계로 바뀌는 중

 + 도메인 네임(Domain Name)
  - 특정 호스트 혹은 다수의 호스트 그룹을 지칭
  - 하나의 IPv4가 할당된 호스트에 다수의 도메인 네임이 매핑될 수 있음
  - 도메인 네임 생성 규칙이 있음


    2. 주소 체계 활용(DNS, DHCP, URL)

    (1)DNS(Domain Name System)
 - 도메인 네임과 IPv4 주소 간의 변환
 - Domain Name Server를 통해 Service가 수행
 - Dynamic DNS 서비스를 통해 유동 IP와 도메인 주소를 연동

    (2)DHCP(Dynamic Host Configuration Protocol)
 - 동적으로 IPv4 주소를 호스트에 할당
 - IPv4 주소의 효율적 사용이 가능
 - 할당이 모두 되어 인터넷 접속이 불가하기도 함

    (3)URL()
 - 호스트 주소와 호스트 내의 자원을 모두 표현하는 방식
 - 프로토콜까지 표현
URL 표현
 - 제일 앞에 자원에 접근할 방법을 정의해 둔 프로토콜
  이름: Http, usenet, telnet 등
 - 프로토콜 이름 다음에 프로토콜 이름을 구분하는 구분자 ":"을 적음
 - IP 혹은 Domain name 정보가 필요한 프로토콜은 : 다음에 //를 적음
 ex)http://www.somehost.com/a.gif


    3. 프로토콜(규약)과 포트 번호

    (1)프로토콜
 - 정보를 주고 받는 주체 간의 약속

    (2)포트 번호
 - 하나의 IPv4 주소 내에 다양한 서비스(프로토콜) 지원
 *- 하나의 포트로 다수의 프로토콜을 지원할 수 없음


    인터넷 주소 체계간 관계 확인

arp -a (Gateway Address)
 - 인터넷 주소가 물리적 주소로 변경된것을 확인
ping 168.126.63.1
 - DNS 서버 핑을 보냄
nslookup www.google.com
 - DNS 서버로부터 주소를 가져오기
