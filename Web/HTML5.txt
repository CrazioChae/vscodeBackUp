1. HTML5 이해
 1. HTML 언어의 역사
  - 1990 물리학자 Tim Berners-Lee가 정의
  - 표준화된 태그로 웹 페이지를 작성하는 언어

 2. HTML5 출현 배경
  - 비표준 기술의 혼재, 웹 브라우저의 비 호환성
   Active-X나 플러그인, 플래시 등 비표준 기술 난립
   브라우저 사이에 HTML 문서와 웹 자원에 대한 심각한 비호환성

  - 인터넷 기기의 다양화
   PC, 모바일 단말기 등에서 모두 웹 사용
   기존의 웹 페이지가 모바일 기기에서 작동하지 않음

  - 새로운 범용 웹 표준의 필요성
   비표준 기술에 의존하는 PC 위주의 기존 웹 방식의 한계
   모바일 기기를 수용할 수 없음
   모바일과 PC에서 동시에 사용할 수 있도록 하는 범용 웹 표준 필요성 대두

  - 인터넷 접속 가능한 다양한 기기
   모바일 기기, 태블릿 PC, 스마트 TV, 게임기 등

 3. HTML5 표준과 의의
  - HTML5 표준 제정
   W3C와 하이퍼텍스트 워킹 그룹

  - 표준에 담긴 내용
   웹 페이지의 구조는 HTML5 태그로, 웹 페이지의 모양은 CSS3로,
    웹 페이지의 행동은 javascript로 분리 개발
   HTML 태그에서 문서의 모양과 관계된 태그나 속성 폐기
   웹 페이지의 플랫폼이나 장치 의존성 제거
   문서 작성의 개념을 넘어 웹 어플리케이션 작성을 지원하는
    자바스크립트 API 표준화

  - 기존 웹 페이지 > 웹 브라우저 > Active-X/플래시 플러그인 > 멀티미디어 지원
  - HTML5로 만든 페이지 > HTML5를 지원하는 웹 브라우저 > 멀티미디어 지원

 4. HTML5의 기능
  - HTML5 전체 기능
   웹 문서 작성을 위한 HTML 태그 셋
   웹 어플리케이션 작성을 위한 API
 
  - HTML5 기능 요약
   웹 폼, 인덱스 데이터베이스, 오디오, 비디오, 파일 입출력, 캔버스, 위치정보 API,
    SVG(Scalable Vector Graphic), 웹 워커, 웹 스토리지, 웹 소켓, 웹 데이터베이스, 
    오프라인 웹 어플리케이션


2. 웹 프로그래밍 개발 과정

 1. HTML5 문서 편집기
  - 텍스트 편집기
   메모장, 한글, 워드 등 편집기 가능
   좋은 편집기 종류 : Atom, Eclipse, Sublime Text 등
   .html인 텍스트 파일로 저장
   텍스트의 기본 문자셋 -UTF-8
   *HTML 파일이나 CSS3, 자바스크립트 파일 모두 UTF-8 코드로 저장되어야 함
  
  - WYSIWYG(What You See Is What You Get) 편집기
   Adobe의 Dreamweaver, CoffeeCup, FCKeditor 등
   HTML5 태그 정보 제공
   출력되는 모습을 보면서 작성 가능
   간단한 오류 체크

 2. HTML5 문서 편집기 사례 : 서브라임 텍스트
  - 설치
  - 실행 및 문서 작성
  - 문서 저장
  - 브라우저에서 HTML 문서 출력

 3. HTML5 태그 검사
  - html5.validator.nu

 4. CSS3 스타일 시트 검사
  - scc-validator.org

 5. 디버깅을 위한 개발자 도구 열기(크롬 브라우저 사례)
  - 클라이언트는 요청을 하고 서버는 응답을 한다.
  - URL은 요청을 만들어내는 주체
  - 서버는 URL을 받아 HTML, CSS, JS를 응답함
  - 요청하는 경우
   파일명, 파일명 + 값
