    01. 컴퓨터(EDPS)


    컴퓨터: 자료를 가공하여 유용한 가치있는 정보로 변환해주는 장치
*데이터 마이닝
 대량의 데이터를 분석하여 일정한 패턴을 찾아내 가치있는 정보를 추출

애니악: 최초의 전자계산기
애드삭: 최초의 프로그램 내장방식(램 있음)
애드박: 내장방식과 2진법(폰 노이만)
유니박: 최초의 상업용 컴퓨터
*컴퓨터의 수동성
 GIGO(Garbage In Garbage Out): 잘못된 입력 > 잘못된 출력


    컴퓨터 세대별 특징
1세대: 일괄처리
2세대: 실시간 처리, 고급언어, 운영체제 도입, 다중 프로그래밍
3세대(비쌈)
 시분할처리(TSS: time sharing system), 경영정보기법(MIS)
4세대
 고밀도 집적회로/ 분산처리 : 개인용 컴퓨터, 마이크로프로세서, 네트워크
5세대
 초고밀도 집적회로/ 인터넷, AI, 퍼지이론, 전문가시스템, 신기술 도입


    자료(데이터)에 따른 분류
디지털 컴퓨터
 숫자/문자(셀 수 있는), 논리회로, 개인용
 프로그래밍 필요(우리가 설치)

아날로그 컴퓨터
 연속적인 자료(셀 수 없는), 증폭회로, 특수 목적용
 프로그래밍 불필요(미리 설치)

하이브리드 컴퓨터: 디지털 아날로그 컴퓨터의 장점을 혼합


    처리 능력에 따른 분류
슈퍼 컴퓨터: 높은 정밀도, 인공위성, 우주항공
메인 프레임: 대규모 시스템, 수백명의 사용자가 동시에 사용
미니 컴퓨터: 중규모 시스템
*워크스테이션: RISC 프로세서를 사용, 네트워크에서 서버역할
데스크톱: CISC 방식


    연산속도 단위(10의 -3승 단위)
ms 밀리 > us 마이크로 > ns 나노 > ps 피코 > fs 펨토 > as 아토
HZ: 1초에 1번, 1KHZ 1초에 1000번
MIPS : 1초에 백만개의 명령어 실행
FLOPS : 초당 부동소수점 연산횟수


    정보처리형태에 따른 분류
일괄처리: 1세대, 한꺼번에 처리
실시간처리: 2세대, 발생하자마자 처리
다중프로그래밍: 한번에 여러개 처리
분산처리: 4세대, 작업을 나눠서 처리
동배간 처리(p2p): 내가 서버/클라이언트
임베디드: 응용프로그램을 탑재, 컴퓨터 기능 수행
*다중처리: 여러개 CPU를 설치해 실행, 빠름
클러스터링: 여러대 컴퓨터를 단일시스템으로 사용
*듀얼: 두개의 CPU가 동시 동작, 한쪽이 고장나면 다른 컴퓨터가 기능 수행
*듀플렉스: 두개 CPU중 단일 동작, 한쪽이 고장나면 다른 컴퓨터가 기능 수행


    자료구성의 단위
비트 -> 니블/4비트 -> 바이트/8비트, 글자를 표현
 -> 워드: 단어를 표현 -> 필드: 워드의 특정항목 -> 레코드: 자료
  -> 블록: 레코드의 모임 -> 파일 -> 데이터베이스(자료의 최종목적)


    자료표현코드
문자표현방식
 아스키코드(데이터 통신용)
 EBCDIC(BCD확장코드)
 유니코드(KS X 1005-1, 모든 문자 2바이트로 표현, 국제표준코드)

에러체크방식
 패리티체크(에러검출만 가능)
 정마크부호(패리티검사가 코드 자체에서 이루어짐)
 해밍코드(에러검출 및 교정을 스스로)
 CRC 체크(우수함)
 BSC 검사(패리티 단점을 보완한 검사)


    숫자표현방식
고정 소수점
 정수표현, 속도가 빠름
 2진연산(부호와 절대치, 1의보수, 2의보수)
 10진연산: 팩, 언팩

부동 소수점
 실수표현, 속도가 느림, 매우 큰수
 부호(1비트), 지수부(7비트), 가수부(소수표현)


    진법

    중앙처리장치(CPU): 레지스터 + 제어장치 + 연산장치
    주변장치 : 입력 + 출력
레지스터
 CPU내에서 가장 빠른 임시 기억장치, 병렬구조
 레지스터의 크기: 한 번에 처리 가능한 데이터(메모리, 워드크기)
 *레지스터 > 캐시 > DRAM > ROM > 하드디스크 > 자기테이프(백업용)

제어장치(CU)
 컴퓨터의 모든 동작을 지시하고 제어
 프로그램 카운터(PC): 다음에 실행할 명령어의 번지를 기억
 명령 레지스터(IR): 현재 실행중인 명령의 내용을 기억
 명령 해독기(Decoder): 명령어 해독(10진수) 변환
 부호기(Encoder): 제어 신호(2진수) 생성하는 회로
 MBR: 데이터를 기억(Buffer)
 MAR: 주소를 기억(Address)

연산장치(ALU)
 가산기: 덧셈
 누산기(ACC, AC): 연산의 중간 결과 기억
 보수기(음수로 변환): 뺄셈하기 위해 사용
 상태 레지스터(PSW): 여러 상태값 기억, 올림수 등

CPU에 영향: 클럭 주파수, 캐시메모리, 워드크기
CPU 성능단위: 클럭(Clock), 밉스(MIPS), 헤르츠(Hz), 플롭스(Flops)
CPU 업그레이드시: 메인보드 지원 확인


    RISC(서버)/CISC(개인)
명령어: 적음 / 많음
주소지정: 간단 / 복잡
레지스터: 많음 / 적음
전기소모: 적음 / 많음
*처리속도: 빠름 / 느림
용도: 서버 / PC


    주기억장치
ROM
 비휘발성 메모리, BIOS(Basic Input Output System)
 글자폰트, 자가진단(POST)

RAM
 휘발성 메모리, 사용중인 프로그램
 데이터 저장

    동적 램(DRAM)/정적 램(SRAM)
재충전: 필요 / 불필요
전기소모: 적음 / 많음
속도: 느림 / 빠름
가격: 저가 / 고가
용도: 주기억장치 / 캐시메모리

플래시 메모리(EEPROM)
 MP3, 디카, 전기 이용, 여러번 읽고 쓰기 가능
MASK ROM: 제조사 단일 기록
PROM: 사용자 단일기록
EPROM: 자외선 이용 다중기록

캐시메모리(SRAM)
 CPU와 주기억장치 사이에 위치, 속도 향상
가상메모리
 하드디스크를 주기억장치처럼 사용
버퍼메모리
 속도차이를 해결하는 임시기억공간(복,붙 사용)
연관 메모리(Associative Memory)
 주소가 아닌 내용의 일부를 이용해 데이터를 가져옴

*펌웨어(Firmware)
 하드웨어 제어, ROM에 저장, 관리
 하드웨어 교체 없이 소프트웨어 업그레이드로 성능을 높일 수 있다.
메모리 용량이 부족할 경우
 램 추가 설치, 불필요한 프로그램 종료, 가상 메모리 크기 조절


    보조기억장치
- 비휘발성, 속도느림, 대용량, 단위당 가격이 저렴
기억용량단위
 2의 10승 = 1024
 Byte > KB > MB > GB > TB > PB(Peta) > EB(Exa)

하드디스크 연결방식
IDE: 2개, EIDE: 4개, SCSI: 7개, 현재는 SATA방식
RAID: 여러개 하드디스크를 한 개의 하드디스크처럼 관리
 미러링: 2개의 디스크에 동일하게 기록
 스트라이핑: 나눠서 기록, 하나라도 손상되면 사용 불가능
 패리티: 장애발생시 패리티를 사용해 복구(가장 많이 사용)

SSD: 반도체를 이용, 부팅속도가 빠름, 높은 내구성, 발열소음전기 적음
파티션: 여러개 논리적 영역으로 나누는 작업
광디스크
 DVD: 대용량 저장
 CD-R: 한번만 기록 가능한 CD
블루레이
 고선명 비디오를 저장
 DVD의 10배 기록, DVD를 블루레이서 사용 불가능(반대는 가능)


    입출력장치
입력장치
 OMR(답안지)
 OCR(공공요금 청구서)
 디지타이저(설계도면 입력)

출력장치
 프린터, 플로터(설계도면 출력)
 표시장치: 모니터

- LCD(액정디스플레이): 유리판에 전기
- 픽셀(화소): 모니터 화면 구성
- 해상도(선명도): 해상도는 픽셀 수로 결정됨


    기타장치
AGP슬롯: 그래픽카드 장착
칩셋: 흐름을 제어, 메인보드의 핵심 장치
롬바이오스: 부팅시 컴퓨터 제어 및 기본 작업을 처리, 기본정보
포트
 직렬포트: 한번에 1비트
 병렬포트(빠름): 한번에 8비트
 USB: 최대 127개 연결
 *- IEEE1394: 가전기기를 컴퓨터에 연결
 - IRDA: 케이블 없이(무선) 적외선으로 연결
 *- HDMI: 영상/음성을 하나의 케이블로 전송


    관련용어
인터럽트: 예기치 않은 상황 > 문제 해결 > 다시 실행
*데드락: 실행중 문제로 멈춰있는 현상
 4가지 조건: 상호배제, 점유와 대기, 비선점, 환형대기
  상호배제: 한 자원에 여러개 프로세서가 하나빼고 전부 대기중인 상태

외부 인터럽트: 입출력 장치, 타이밍 장치, 전원
내부 인터럽트: 잘못된 명령이나 데이터
가장 높은 우선순위 인터럽트: 정전
가장 낮은 우선순위 인터럽트: SVC <- 사용자 의도 인터럽트

*채널: CPU대신 입출력을 관리
채널 종류: 셀럭터(고속), 멀티플렉서(저속)
*DMA(Direct Memory Access(직접메모리접근))

    한글 Windows10 특징

에어로 쉐이크: 제목표시줄을 흔들면 나머지창이 최소화


    라이브러리(문서, 음악, 사진, 비디오)
 - 나눠진 자료를 한 곳에서 보고 정리할 수 있게 하는 가상의 폴더

 라이브러리 포함된 폴더나 파일을 제거하면 원본이 삭제됨

    03. 네트워크/인터넷

 AP, 무선랜카드
 *안테나

    프로토콜
 - 네트워크에서 서로 다른 컴퓨터들간 
 정보 교환을 할 수 있게 해주는 통신 규칙

HTTP: 하이퍼텍스트 전송 프로토콜

 *전송 계층에 해당

 *네트워크 계층에 해당

프로토콜: 통신규칙

 (C클래스: 255.255.255.0)

    네트워크 관련 용어

  아날로그 -부호화-> 디지털 -복호화-> 아날로그

    4. 멀티미디어

    
    멀티미디어
텍스트, 그래픽, 사운드 매체를 디지털로 통합하여 전달(비동기 방식)
멀티미디어 특징: 디지털화, 쌍방향성, 비선형성, 통합성
*쌍방향성
  정보제공자와 사용자간 상호작용에 의해 데이터가 전달되는 방식


    멀티미디어 관련용어
하이퍼텍스트: 문서와 문서가 연결
하이퍼미디어: 하이퍼텍스트에 멀티미디어를 합친것
노드: 하이퍼텍스트에서 각 문서에 연결된 페이지
앵커: 노드에서 다른 노드로 넘어가게 하는 키워드
하이퍼링크: 노드와 노드의 연결점
저작 소프트웨어
 간단한 메뉴 조작만으로 멀티미디어 자료를 만들수 있음
AVI: 윈도우즈 표준 동영상
MPEG-2: HD TV
MPEG-4: IMT 2000 영상정보
MPEG-7: 정보검색, 전자상거래
*MPEG-21: 디지털 컨텐츠 제작, 유통, 보안등 전 과정을 관리
H.264: 위성DMB 비디오기술의 표준, MPEG-4 형식을 따름
VCS: 화상회의 시스템
VOD: 주문형 비디오 시스템, 시간과 장소에 구애받지 않음
사운드카드 기록용량: (샘플링시간*주파수*비트수*채널수/8)Byte
비디오 오버레이보드: 외부비디오신호를 컴퓨터화면과 같이 표현
샘플링: 아날로그(음성)를 디지털신호로 변환
가상현실: VR
스트리밍: 다운로드하면서 동시에 재생(실시간)
CAI: 컴퓨터를 이용한 수업 방식
압축파일: 파일크기 줄임, 전송시간 절약, 디스크 공간 효율적으로 사용
MOD: 실시간 음원방송(멜론, 지니, 바이브)
비트맵: 실제 이미지, 용량이 큼, 확대하면 계단현상
MIME: 멀티미디어 전자우편(동영상, 음악, 카드메일)
백터방식: 테두리가 거칠어지지 않음(일러스트)
PACS: 원격진료(독도, 울릉도)
JPEG: 정지영상 국제표준(손실, 무손실)
MPEG: 동영상 국제표준(손실 압축만 지원)
GIF: 256색상(무손실 지원, 애니메이션)
PNG: 웹상 최적의 이미지표현
WAVE: 실제 소리저장, 용량이 매우 큼
MP3: 1/12로 압축, MPEG-1 압축기술
MIDI: 악기 표준 프로토콜(*소리 저장 안됨)
IMAP: 이메일 제목만 다운로드
FLAC: 압축 시 스트림 손실이 없음
AIFF: 애플
WAV: 윈도우 표준
MHGC: ISO 위원회, 멀티미디어 콘텐츠 등을 기술하는 언어의 표준



    5. 정보통신


    통신망 운용방식
중앙 집중식: 모든 처리는 중앙컴퓨터 / 입출력은 단말기가
클라이언트/서버: 자료를 요청 / 자료를 제공(분산처리 방식)
동배간처리: p2p방식, 서버도 되고 클라이언트도 됨


    통신망의 종류
LAN(근거리)
VAN(부가가치)
MAN(도시권)
WAN(전세계)
IMT-2000: 한대의 휴대폰으로 전 세계 어디서나 사용하는 규격
ISDN(종합정보)
 문자, 음성, 동영상 등 여러 데이터를 통합해
  하나의 통신 회선으로 전송
B-ISDN: ISDN에 광통신을 결합함
ADSL
 다운로드와 업로드 속도가 달라
  비대칭 디지털 가입자 라인이라 함


    통신망의 구성형태
성형
 중앙노드의 제어
 처리능력, 신뢰성
 고장발견, 유지보수확장
링형
 포인트 투 포인트(1대1)
 단말기 추가/제거/기밀보호 어려움
 하나의 노드고장이 전체에 영향
버스형
 설치가 용이, 경제적
 컴퓨터간 접속은 백본으로 이뤄짐
망형
 응답시간이 빠름
 많은 양의 통신을 필요로 할 때 사용
 *필요한 회선수 N(N-1)/2
계층형(트리구조)
 중앙부터 일정지역까지는 하나의 회선으로 연결(부모자식관계)


    정보전송방식
단향: SIMPLEX
반이중: HALF DUPLEX, 양쪽 모두 송수신, 동시는 불가능(무전기)
전이중: FULL DUPLEX, 양쪽 모두 동시에 송수신(전화)


    전기장치
UPS: 정전에 대비
AVR: 항상 일정한 출력
CVCF: 전압 일정, 주파수 일정


    정보통신 관련용어
N스크린: 멀티미디어 콘텐츠를 여러 스마트기기에 연결
WIFI: 무선랜의 표준
NFC: 비접촉 근거리 무선통신 모듈
와이브로: 무선 광대역, 이동하면서 인터넷 사용
핫스팟: 자신의 인터넷을 일정공간 안에서 다른 이들에게 공유
사물 인터넷(IOT)
 인간>사물 / 사물>사물
 통신비용 증가, 보안기능 약해짐
UX
 기기를 사용하며 느끼는 총체적인 경험을 바탕으로
  삶의 질을 높이려는 접근방법
SNS: 서로의 관심사를 공유
앱스토어: 스마트폰에 설치할 프로그램을 판매
테더링: 다른 기기에 인터넷을 연결
클라우드 서비스
 자신의 콘텐츠를 서버에 두고
  어느 기기에든 사용할 수 있게 하는 기술
3D 프린트
 XYZ축 작성된 3D도면을 목표한 형태와 똑같이 나오게 하는것
 (적층형, 절삭형)



    6. 보안
    
    
    정보사회와 보안
정보가 원동력이 되는 사회
능률과 생산성이 증대
새로운 인간관계문화 형성
시간과 공간제약 극복
상품 생명력은 짧음


    정보사회의 부작용
정보의 과다로 인한 혼란, 계층간 정보차이발생
비인간화, 사회적 혼란초래
정보소외현상
VDT증후군: 장시간 모니터를 보며 생겨나는 증상(불안감)


저작권법
 저작자의 권리를 보호, 공정한 이용을 도모해 산업발전에 이바지


    저작권 보호기간(2013년 7월 1일부터)
시작일: 사망일 또는 다음해 1월 1일부터
사망 후 70년간 존속함
공동저작물의 경우 마지막으로 사망한 자가 기준
무명 또는 이명일 경우 공표된 시점을 기준


    보안위협형태
스푸핑: 웹사이트 구축 > 정보를 빼냄
스니핑: 패킷을 엿보면서 아이디, 패스워드를 빼냄
스파이웨어: 동의 없이 사용자 정보를 수집
혹스: 악성코드인 것처럼 가장
DDoS: 여러 PC가 동시에 특정사이트를 공격
웜
 네트워크를 통해 연속적으로 자신을 복제
 시스템부하를 일으켜 다운
메모리 해킹: 램에 있는 데이터를 위조/변조
스미싱
 SMS와 피싱의 합성어
 수신자가 클릭하면 금융정보를 빼냄
백도어: 보안이 제거된 비밀통로
파밍: 허위 사이트로 유도해 금융정보를 빼냄
피싱: 가짜 이메일을 보내고 개인금융정보를 빼냄


    방화벽: 외부의 불법침입으로부터 막아줌(내부자산을 보호)
역추적기능이 있어 외부침입자의 흔적을 찾을 수 있음
 (내부해킹은 못막음)


    암호화
 지정한 수신자 이외에는 내용을 알 수 없도록 변환
 (도청, 부정 접근)

비밀키
 동일한 키, 암호화하고 복호화 함
 DES, 속도가 빠르며, 알고리즘이 단순, 크기가 작음
 키의 수가 많아짐

공개키
 서로 다른 키로 데이터를 암호화하고 복호화 함
 암호화할 때는 공개키, 복호화할 때는 비밀키
 RSA, 키의 개수가 적고 속도가 느림
 알고리즘 복잡, 파일 크기가 크다


    보안 요건
기밀성: 허가된 사용자만 이용
무결성: 허가된 사용자만 수정 가능
가용성: 언제든지 사용 가능
인증: 보낸 사람의 신원확인
부인방지: 송, 수신 사실의 증거를 제공


    컴퓨터 범죄유형
불법복제, 사기, 해킹
개인정보 유출, 음란물 배포, 바이러스 유포


    컴퓨터범죄예방 및 대책
 *백신 기능: 검사, 치료, 예방
대책
 보안 프로그램 보급
 보안교육 실시
 비밀번호 변경
 백신프로그램설치
 자동업데이트 설정
 불분명한 이메일은 삭제
 중요한 자료는 백업


    바이러스 종류
파일바이러스: 실행파일감염
부트바이러스: 부트섹터를 손상
부트/파일: 파일과 부트섹터, Invader, 에볼라
*매크로바이러스
  엑셀문서를 감염시키는 바이러스
  멜리사, Laroux
클러스터: 감염된 디스크에서 실행하면 동시에 감염
은닉바이러스: 메모리에 상주
스크립트: 파일 안에 스크립트
폭탄바이러스: 일정한 조건을 만족
*공유폴더의 속성을 숨김으로 해도
 네트워크 감염을 방지할 수 없다
 (예방을 위해선 읽기전용으로 변경)
