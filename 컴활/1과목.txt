1. 컴퓨터
자료를 가공하여 유용한 가치있는 정보로 변환해주는 장치
데이터 마이닝: 대량의 데이터를 분석하여 일정한 패턴을 찾아내 가치있는 정보를 추출
애니악 : 최초의 전자계산기
애드삭 : 프로그램 내장방식 램 있음
애드박 : 내장방식과 2진법(폰 노이만)
유니박 : 최초의 상업용 컴퓨터(GIGO 잘못된 입력 > 잘못된 출력)

1세대 : 일괄처리
2세대 : 실시간 처리, 고급언어, 운영체제 도입
3세대 : 시분할처리(TSS time sharing system), 경영정보기법(MIS)
4세대 : 고밀도 집적회로 / 분산처리 : 개인용 컴퓨터, 마이크로프로세서
5세대 : 초고밀도 집적회로 / 인터넷, AI, 퍼지이론, 전문가시스템, 신기술 도입

디지털 컴퓨터 : 숫자/문자, 논리회로, 개인용
아날로그 컴퓨터 : 연속적인 자료, 증폭회로, 특수 목적용
하이브리드 컴퓨터 : 디지털 아날로그 컴퓨터의 장점을 혼합

처리 능력에 따른 분류
슈퍼 : 높은 정밀도, 인공위성, 우주항공
메인 : 대규모 시스템, 수백명의 사용자가 동시에 사용
미니 컴퓨터 : 중규모 시스템
워크스테이션 : RISC 프로세서를 사용, 서버역할
데스크톱 : CISC 방식

ms 밀리 / us 마이크로 / ns 나노 / 
ps 피코 / fs 펨토 / as 아토

HZ 1초에 1번, 1KHZ 1초에 1000번
MIPS : 1초에 백만개의 명령어 실행
FLOPS : 초당 부동소수점 연산횟수

정보처리형태에 따른 분류
일괄처리 : 1세대, 한꺼번에
실시간처리 : 2세대, 발생하자마자
다중프로그래밍 : 한번에 여러개
분산처리 : 4세대, 작업을 나눠서 처리
동배간 처리 : 내가 서버/클라이언트
임베디드 : 응용프로그램을 탑재, 컴퓨터 기능 수행
*다중처리 : 여러개 CPU를 설치해 실행
클러스터링 : 여러대 컴퓨터를 단일시스템으로 사용
듀얼 : 두개의 CPU가 동시 동작, 하나가 고장나면 다른 컴퓨터가 기능 수행
듀플렉스 : 두개 CPU중 단일 동작, 한쪽이 고장나면 다른 컴퓨터가 기능 수행

중앙처리장치(CPU): 레지스터 + 제어장치 + 연산장치, 주변장치 : 입력 + 출력
레지스터
 - CPU내에서 가장 빠른 임시 기억장치, 병렬구조
 - 레지스터의 크기: 한 번에 처리 가능한 데이터(메모리, 워드크기)
 - 레지스터 > 캐시 > DRAM > ROM > 하드디스크 > 자기테이프(백업용)

제어장치(CU)
 - 컴퓨터의 모든 동작을 지시하고 제어
 - 프로그램 카운터: 다음에 실행할 명령어의 번지를 기억, PC
 - 명령 레지스터: 현재 실행중인 명령의 내용을 기억, IR
 - 명령 해독기(Decoder): 해독(10진수로 변환)
 - 부호기(Encoder): 해독 > 제어 신호(2진수로 변환)
 - MBR: 데이터를 기억(Buffer)
 - MAR: 주소를 기억(Address)

연산장치(ALU)
 - 가산기: 덧셈, 누산기: 연산의 중간 결과 기억
 - 보수기(음수로 변환): 뺄셈하기 위해 사용
 - 상태 레지스터: 여러 상태값 기억(PSW)

CPU에 영향: 클럭 주파수, 캐시메모리, 워드크기
CPU 성능단위: 클럭, 밉스, 헤르츠, 플롭스
CPU 업그레이드시: 메인보드 지원 확인

RISC/CISC
명령어: 적음 / 많음
레지스터: 많음 / 적음
주소지정: 간단 / 복잡
전기소모: 적음 / 많음
처리속도: 빠름 / 느림
용도: 서버 / PC

    주기억장치
ROM: 비휘발성, BIOS, POST
RAM: 휘발성, 사용중인 프로그램, 데이터 저장

DRAM/SRAM
재충전: 필요 / 불필요
전기소모: 적음 / 많음
가격: 저가 / 고가
용도: 주기억장치 / 캐시메모리

플래시 메모리(EEPROM): MP3, 디카, 전기 이용, 여러번 읽고 쓰기 가능
MASK ROM(제조사 단일 기록), PROM(사용자 단일기록), EPROM(자외선 이용 다중기록)

캐시메모리(SRAM): CPU와 주기억장치 사이에 위치, 속도 향상
가상메모리: 하드디스크를 주기억장치처럼 사용
버퍼메모리: 속도차이를 해결하는 임시기억공간(복,붙 사용)
연관 메모리(Associative Memory): 주소가 아닌 내용의 일부를 이용해 데이터를 가져옴
*펌웨어(Firmware): 하드웨어 제어, ROM에 저장, 관리
 - 하드웨어 교체 없이 소프트웨어 업그레이드로 성능을 높일 수 있다.
메모리 용량이 부족할 경우 : 램 추가 설치, 불필요한 프로그램 종료

보조기억장치: 비휘발성, 속도느림, 대용량, 단위당 가격이 저렴
기억용량단위 2의 10승 = 1024
Byte > KB > MB > GB > TB > PB > EB

프로토콜: 네트워크에서 서로 다른 컴퓨터들간
 정보 교환을 할 수 있게 해주는 통신 규칙
TCP/IP: 인터넷 표준 프로토콜
HTTP - 하이퍼텍스트 전송 프로토콜 
TCP - 패킷 단위로 분류, 흐름제어, 에러 유무 검사, *전송 계층에 해당
IP - 패킷 주소를 해석, 경로를 결정, *네트워크 계층에 해당
ARP - IP > 물리적주소로 변경(거꾸로: RARP)
DHCP - IP주소를 자동으로 할당, 동적으로 IP주소를 부여
ICMP - 네트워크기기의 상태를 확인하기 위해 보내는 인터넷 제어메시지
(ACK: 긍정, NAK: 부정)
UDP: 한 컴퓨터에서 다른 컴퓨터로 데이터를 전송하는 프로토콜

 - 전자우편: SMTP(전송), POP3(수신), MIME(멀티미디어 전자우편)

 - 참조와 숨은 참조의 차이: 숨은 참조는 수신자가 표시되지 않음

네트워크 명령어
- Ping: 원격컴퓨터가 네트워크에 정상적 작동여부 확인
- IPCONFIG: 현재 PC의 물리적 주소, IP주소, Subnet Mask, 게이트웨이 확인
- Tracert ip주소: 상대방 컴퓨터 경로 추적
- NETSTATE: 내 컴퓨터 연결된 다른 컴퓨터의 정보를 확인
- NBTSTATE: 중복 IP주소로 인해 충돌하는 지점을 확인
- NET VIEW: 특정 컴퓨터에 공유된 정보를 확인
- NSLOOKUP: URL주소로 IP주소를 확인, DNS 동작여부를 확인
